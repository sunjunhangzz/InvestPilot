# web 开发指南

## 定位

`web` 是第一版系统的用户界面，同时承载 Next.js API Routes。

第一版不单独启动独立后端服务，前端页面和 API 放在同一个 Next.js 项目里。

## 页面范围

第一版页面：

```text
/dashboard          首页仪表盘
/recommendations    今日推荐
/stocks/[code]      股票详情
/watchlist          观察池
/settings           系统设置
```

## 开发顺序

建议按这个顺序开发：

1. 搭 Next.js 项目骨架。
2. 创建全局布局和导航。
3. 实现静态页面结构。
4. 接入 mock 数据。
5. 接入真实 API。
6. 增加加载、空数据、失败状态。
7. 增加图表展示。

## 页面原则

- 首页展示系统状态，不堆太多表格。
- 推荐列表只展示核心字段。
- 股票详情页展示完整指标和图表。
- 观察池强调连续跟踪状态。
- 设置页只放第一版必要配置。

## 状态处理

每个页面都要处理：

- loading
- empty
- error
- success

长任务按钮要处理：

- idle
- pending
- running
- success
- failed

## 图表

第一版只做三类图表：

- 近 60 日收盘价折线图
- 近 60 日成交额柱状图
- 因子评分条形图

不要第一版就做复杂大屏。
