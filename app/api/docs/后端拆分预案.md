# 后端拆分预案

## 当前方案

第一版使用 Next.js API Routes。

```text
web 页面
  ↓
Next.js API Routes
  ↓
Python worker
  ↓
SQLite
```

## 后续拆分条件

当出现以下问题时，可以拆出独立后端：

- 长任务越来越多。
- 任务依赖关系复杂。
- 多用户权限需求出现。
- SQLite 升级为 PostgreSQL。
- 需要 Redis 队列。
- 需要远程部署。

## 拆分后方案

可能的技术路线：

```text
前端：Next.js
后端：Fastify / NestJS
任务队列：BullMQ / Redis
数据库：PostgreSQL
Python Worker：独立任务进程
```

## 迁移原则

- 不提前引入复杂度。
- 先保持本地单机可运行。
- 拆分时保留原有 API 契约。
- 数据库 schema 优先保持兼容。
