# worker 开发指南

## 定位

`worker` 是 Python 数据和计算层。

它不负责页面展示，只负责：

- 获取数据
- 清洗数据
- 写入数据库
- 计算因子
- 生成推荐
- 维护观察池
- 生成基础 AI 报告

## 开发顺序

建议按这个顺序开发：

1. `init_db.py`
2. `update_stocks.py`
3. `update_prices.py`
4. `calc_factors.py`
5. `run_screening.py`
6. `update_watchlist.py`
7. `generate_report.py`

## Python环境

使用 Python 3.12。

依赖统一写在：

```text
app/worker/requirements.txt
```

创建环境：

```bash
cd /Users/sjh/Desktop/A股AI投研系统/app/worker
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

说明：

- `.venv` 是本地环境，不提交到 Git。
- AkShare 依赖网络和上游接口，安装或运行失败时需要记录错误。
- 第一版依赖只保留数据采集、数据处理、环境变量读取和 HTTP 调用需要的包。

## 脚本约定

每个脚本都应支持命令行执行。

示例：

```bash
python scripts/update_prices.py --run-id 20260704-manual-001
```

每个脚本都需要：

- 读取 shared/config.json。
- 写入 system_tasks 状态。
- 失败时返回非 0 exit code。
- 失败时写入错误信息。

## 数据写入原则

- 使用事务。
- 空数据不覆盖旧数据。
- 重复数据使用 upsert。
- 缺失字段记录日志。
- 写入前校验 trade_date。

## 不做

第一版 worker 不做：

- 高频行情
- 实时盘口
- 复杂回测
- 多 Agent 编排
- 自动调参
- 新闻情绪评分
