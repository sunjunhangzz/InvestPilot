# worker 开发指南

## 定位

`worker` 是 Python 数据和计算层。

它不负责页面展示，只负责：

- 获取数据
- 清洗数据
- 写入数据库
- 计算因子
- 生成推荐
- 维护观察池
- 生成基础 AI 报告

## 开发顺序

建议按这个顺序开发：

1. `init_db.py`
2. `update_stocks.py`
3. `update_prices.py`
4. `calc_factors.py`
5. `run_screening.py`
6. `update_watchlist.py`
7. `generate_report.py`

## 脚本约定

每个脚本都应支持命令行执行。

示例：

```bash
python scripts/update_prices.py --run-id 20260704-manual-001
```

每个脚本都需要：

- 读取 shared/config.json。
- 写入 system_tasks 状态。
- 失败时返回非 0 exit code。
- 失败时写入错误信息。

## 数据写入原则

- 使用事务。
- 空数据不覆盖旧数据。
- 重复数据使用 upsert。
- 缺失字段记录日志。
- 写入前校验 trade_date。

## 不做

第一版 worker 不做：

- 高频行情
- 实时盘口
- 复杂回测
- 多 Agent 编排
- 自动调参
- 新闻情绪评分
