# 本地启动说明

## 目标

说明如何在本机启动第一版开发环境。

当前阶段已经具备：

- Python worker 虚拟环境
- Next.js web 骨架
- shared 配置和路径解析

数据库初始化、数据采集和筛选脚本会在后续任务中逐步补齐。

## 前置环境

确认 Python 和 Node.js 可用：

```bash
python3 --version
node --version
npm --version
```

当前项目使用：

```text
Python 3.12
Node.js 24
npm 11
```

## 环境变量

复制示例配置：

```bash
cd /Users/sjh/Desktop/A股AI投研系统
cp .env.example .env
```

`.env` 只保存在本地，禁止提交到 Git。

第一版基础规则筛选不依赖 AI Key。只有启用 AI 报告时，才需要填写对应模型的 API Key。

## 启动 worker 环境

```bash
cd /Users/sjh/Desktop/A股AI投研系统/app/worker
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

检查依赖：

```bash
python -c "import akshare, pandas, numpy, requests, dotenv; print('worker ok')"
```

## 启动 web

```bash
cd /Users/sjh/Desktop/A股AI投研系统/app/web
npm install
npm run dev -- --hostname 127.0.0.1
```

浏览器打开：

```text
http://127.0.0.1:3000/dashboard
```

## 常见问题

### npm run dev 端口被占用

换一个端口：

```bash
npm run dev -- --hostname 127.0.0.1 --port 3001
```

### AkShare 安装失败

优先检查网络，然后重新执行：

```bash
python -m pip install -r requirements.txt
```

### 不要提交本地运行产物

以下内容只保存在本地：

```text
.env
app/worker/.venv/
app/web/node_modules/

```
