# 任务机制

## 为什么需要任务机制

数据更新、因子计算、推荐生成和 AI 报告可能运行几十秒到几分钟。

前端不能一直等待 API 返回，否则容易超时或让用户误以为系统卡死。

## 执行方式

第一版采用异步任务：

```text
POST /api/tasks/update-data
  ↓
创建任务，返回 task_id
  ↓
后台执行 Python worker
  ↓
前端轮询任务状态
```

## 任务状态

```text
pending
running
success
failed
cancelled
```

## API 设计

```text
POST /api/tasks/update-data
POST /api/tasks/run-screening
POST /api/tasks/generate-report
GET  /api/tasks/:task_id
GET  /api/runs/latest
GET  /api/recommendations/today
GET  /api/watchlist
```

## 串行执行

第一版所有写入任务串行执行。

禁止同时运行：

- 两个数据更新任务
- 数据更新和筛选任务
- 两个筛选任务
- 筛选和观察池更新任务

## 失败处理

- 任务失败必须写入 `system_tasks.error_message`。
- 前端必须展示失败原因。
- 数据源失败不能覆盖已有可用数据。
- AI 失败不能影响基础推荐。
- 用户可以手动重试失败任务。

## 非交易日处理

周末和节假日运行时：

- 系统提示当前可能是非交易日。
- 默认使用最近一个交易日数据。
- 不强行生成新的交易日结果。
