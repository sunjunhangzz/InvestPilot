# 04-日志代码规范

## 日志格式

日志使用 JSON Lines。

每行一条 JSON。

必须字段：

```text
time
level
module
message
```

推荐字段：

```text
task_id
run_id
trade_date
context
```

## 日志等级

```text
DEBUG
INFO
WARN
ERROR
```

## 日志位置

```text
data/logs/
```

文件：

```text
worker.log
data_source.log
screening.log
ai.log
api.log
```

## Python日志

Python 使用标准 `logging` 模块封装统一 logger。

禁止每个脚本自己随意配置日志格式。

## TypeScript日志

API Routes 写 `api.log`。

前端页面不写本地文件日志，只展示 API 返回的错误摘要。

## 必须记录

- 任务开始
- 任务结束
- 任务失败
- 数据源返回空
- 单只股票采集失败
- 推荐数量
- 观察池状态变化
- AI 调用失败
- 子进程退出码

## 禁止记录

禁止写入日志：

- API Key
- 密码
- 完整 `.env`
- 任何敏感密钥

## 日志相关注释

日志字段含义不直观时，需要在代码中写注释。

必须注释的场景：

- 为什么某个错误只写摘要到 `system_tasks`
- 为什么详细堆栈只写入 `data/logs`
- 为什么某些字段不能进入日志
- 日志脱敏逻辑
- 日志滚动或保留策略

日志脱敏逻辑必须有注释说明。
