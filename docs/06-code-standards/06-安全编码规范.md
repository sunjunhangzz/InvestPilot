# 06-安全编码规范

## 密钥

API Key 只能放在：

```text
.env
```

禁止：

- 写入代码
- 写入日志
- 写入文档
- 提交到 Git

## .env

必须提供：

```text
.env.example
```

`.env.example` 只写变量名，不写真实值。

## SQL安全

所有 SQL 必须参数化。

禁止字符串拼接用户输入。

## API安全

第一版本地使用，不做多用户权限。

但仍然要：

- 校验 API 参数
- 限制任务类型
- 拒绝未知脚本名
- 不允许前端传任意 shell 命令

## 子进程安全

API Routes 调 worker 时：

- 只允许调用白名单脚本
- 参数必须校验
- 不拼接 shell 命令
- 使用参数数组调用子进程

## AI安全

AI 只做解释，不做最终买卖决策。

传给 AI 的内容必须是结构化摘要。

禁止把 API Key、系统路径、`.env` 内容传给模型。

## 数据文件

禁止提交：

```text
data/*.sqlite
data/cache/
data/raw/
data/logs/
reports/
.env
.venv/
node_modules/
```
