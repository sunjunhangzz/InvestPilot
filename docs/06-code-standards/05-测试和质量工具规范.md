# 05-测试和质量工具规范

## 目标

第一版测试目标：

- 确认数据链路可跑通
- 确认推荐逻辑可解释
- 确认失败可定位
- 确认页面基本可用

## Python测试

优先测试：

- 配置读取
- 数据库连接
- schema 校验
- MA20 / MA60 计算
- 因子评分
- 推荐数量控制
- 观察池状态流转

建议使用：

```text
pytest
```

## TypeScript测试

第一版至少保证：

- `npm run lint` 通过
- 类型检查通过
- API 返回结构稳定

后续可加：

```text
Playwright
```

## 质量工具

Web：

```text
ESLint
Prettier
TypeScript
```

Python：

```text
ruff 或 black
pytest
```

第一版可先把工具写入计划，不必一次全部配置复杂。
