# 00-项目基建

## 任务 00.1：确认目录结构

所属子项目：全局

前置依赖：无

开发内容：

- 确认 `app/web`
- 确认 `app/worker`
- 确认 `app/shared`
- 确认 `data`
- 确认 `reports`
- 确认 `docs`

验收标准：

- 所有目录存在。
- 文档索引能说明目录用途。

风险点：

- 目录含义不清，后续文件乱放。

## 任务 00.2：确认 shared 配置

所属子项目：shared

前置依赖：00.1

开发内容：

- 检查 `app/shared/config.json`
- 检查 `app/shared/schema.json`
- 确认数据库路径
- 确认报告路径
- 确认推荐数量
- 确认因子权重

验收标准：

- `config.json` 是合法 JSON。
- `schema.json` 是合法 JSON。
- 数据库路径指向 `data/a_stock_research.sqlite`。
- 所有相对路径都按项目根目录解析。
- 因子权重总和为 1。

风险点：

- 相对路径从不同工作目录读取时可能失效。

必须通过统一路径解析工具解决该风险，禁止在 web 和 worker 中各自手写路径拼接。

所属子项目：worker

前置依赖：00.2

开发内容：

## 任务 00.3a：创建 worker 源码骨架

所属子项目：worker

前置依赖：00.3

开发内容：

- 创建 `app/worker/src/config/`
- 创建 `app/worker/src/db/`
- 创建 `app/worker/src/data_sources/`
- 创建 `app/worker/src/factors/`
- 创建 `app/worker/src/screening/`
- 创建 `app/worker/src/watchlist/`

## 任务 00.5：创建基础工程文件

所属子项目：全局

前置依赖：00.1

开发内容：

- 创建根目录 `.gitignore`
- 创建根目录 `.env.example`
- 创建 `app/worker/requirements.txt`
- 创建本地启动说明

验收标准：

- `.gitignore` 排除 `node_modules`、`.venv`、数据库文件、缓存、日志、`.env`
- `.env.example` 不包含真实密钥
- `requirements.txt` 包含第一版 Python 依赖
- 新开发者能根据文档启动项目

风险点：

- 误把 API Key、数据库、缓存文件提交进版本管理

## 任务 00.6：创建日志目录约定

所属子项目：全局

前置依赖：00.1

开发内容：

- 创建 `data/logs/`
- 约定 worker 日志文件位置
- 约定 API 日志和任务错误记录方式

验收标准：

- 数据采集失败能在日志中定位
- AI 调用失败能在日志中定位
- 任务失败能在 `system_tasks.error_message` 中看到摘要

风险点：

- 没有日志时，数据源或任务失败难以排查

## 任务 00.7：创建日志工具约定

所属子项目：worker / web

前置依赖：00.6

开发内容：

- worker 统一日志格式。
- API 统一日志格式。
- 日志写入 `data/logs/`。
- 日志使用 JSON Lines 格式。

验收标准：

- worker 能写 `worker.log`。
- API 能写 `api.log`。
- 日志包含 `task_id`、`run_id`、`level`、`module`、`message`。

风险点：

- 各模块各写各的日志格式，后续难以收集。
