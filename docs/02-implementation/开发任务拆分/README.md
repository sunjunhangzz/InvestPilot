# 开发任务拆分

## 目标

把第一版开发计划拆成可以逐个执行、逐个验收的小任务。

每个任务都包含：

- 目标
- 所属子项目
- 前置依赖
- 开发内容
- 验收标准
- 风险点

## 执行顺序

建议严格按顺序推进：

```text
00-项目基建
01-数据库
02-数据采集
03-因子和推荐
04-观察池
05-API任务层
06-前端页面
07-AI基础报告
08-本地联调
09-第2阶段收尾
10-第3阶段自动化和推送
11-第4阶段基本面增强
12-第5阶段复盘闭环
```

## 执行原则

- 一个任务完成后再进入下一个任务。
- 每个任务都要能独立验证。
- 不在当前任务中提前实现后续阶段能力。
- 如果发现文档和代码冲突，先更新文档再继续实现。
- 优先保证主链路跑通。
- 数据采集先单只股票测试，再批量运行。
- 前端先用 mock 数据搭页面，再接真实 API。
- 长任务必须有任务锁和失败重试。
- AI 输入必须受控，避免 token 成本失控。
