## 推送时间

| 时间 | 名称 | 内容 | 触发 |
|---|---|---|---|
| 08:00 | 早盘报告 | 昨日推荐摘要 + 第一名 AI 总结 | `run_analysis.sh && send_report.py --type=morning` |
| 12:00 | 午间报告 | 推荐列表 + 上午盘涨跌幅排序 | 先 `update_prices.py --codes=<推荐列表>`（仅推荐股，~15秒），再 `send_report.py --type=noon` |

## 推送方式

**默认：企业微信群机器人** — 腾讯官方，免费无限制，支持 Markdown 表格排版。

API：`POST {webhook_url}` `{ "msgtype": "markdown", "markdown": { "content": "..." } }`

**备选：PushPlus**（200 条/天）、**Server酱**（5 条/天）。通过 `wechat.provider` 切换。

| Provider | 配置 key | 免费额度 | 格式 |
|---|---|---|---|
| wecom | `wechat.webhook_url` | 无限制 | Markdown |
| pushplus | `wechat.token` | 200/天 | 纯文本 |
| serverchan | `wechat.token` | 5/天 | 纯文本 |

## 配置存储

settings 表新增 key：

| key | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `wechat.enabled` | bool | false | 是否启用微信推送 |
| `wechat.provider` | string | "wecom" | 推送渠道：wecom / pushplus / serverchan |
| `wechat.webhook_url` | string | "" | 企业微信机器人 Webhook URL |

### 任务 10.1：send_report.py 推送脚本

所属子项目：worker

前置依赖：03-07 全部完成

**开发内容**：

- 创建 `app/worker/scripts/send_report.py`
- 支持 `--type=morning|noon` 和 `--dry-run`（只写日志不发推送）
- morning 模式：从最新 success run 取推荐摘要 + 首名 AI 总结
  - **无数据时发提醒**：「今日暂无推荐数据，请手动运行筛选」
- noon 模式：先对推荐股票执行增量行情更新（仅 ~50 只，~15 秒），再查 daily_prices 最新两行计算上午涨跌幅
  - **数据未更新时提醒**：「行情数据未更新，建议手动刷新行情后重新筛选」
- 查 settings 表获取 `wechat.enabled` / `wechat.provider` / `wechat.webhook_url`
- 根据 provider 调对应 API（默认 wecom：POST webhook_url + msgtype=markdown）
- 推送成功/失败写入 `data/logs/notification.log`

**验收标准**：

- 早盘推送：含推荐摘要 + 第一名 AI 总结
- 午间推送：含推荐列表 + 涨跌幅排序
- wechat.enabled=false / token 为空时跳过不报错
- `--dry-run` 只写日志不真发
- 企业微信 / PushPlus / Server酱 三个 provider 可切换

### 任务 10.3：launchd 定时任务

所属子项目：全局

前置依赖：10.1

**开发内容**：

- 创建 `scripts/morning_push.sh`：`bash run_analysis.sh && python send_report.py --type=morning`
- 创建 `scripts/noon_push.sh`：先 `update_prices.py --codes=<推荐列表>`（仅推荐股，~15秒），再 `python send_report.py --type=noon`
- 创建 plist 模板文件（`scripts/*.plist`，文档性质）
- 提供安装说明

**验收标准**：

- plist 模板可被 launchd 加载
- 手动运行脚本正常

**使用须知**：

- 定时推送需要**电脑保持开机**（睡眠时 launchd 不触发，唤醒后自动补跑）
- 如果电脑关机错过了推送时间，可以手动运行 `bash scripts/morning_push.sh`
- 不依赖网络连接（只在本地跑），但推送需要外网

### 任务 10.4：午间涨跌幅计算（含在 10.1 中）

所属子项目：worker

**开发内容**：

- 对每只推荐股票，查询 daily_prices 最近两行
- 计算 `(latest.close - prev.close) / prev.close * 100`
- 按涨跌幅排序后组装推送内容

**验收标准**：

- 涨跌幅计算正确
- 按涨幅排序展示
