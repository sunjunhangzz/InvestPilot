## 14.1 数据字典 & 字段说明

### 背景

用户反馈：系统中大量金融术语（PE/PB/ROE/Sharpe/因子评分/换手率等）对非金融从业者不友好，不知道每个指标的含义、好坏标准、数据来源。

### 产品需求

**目标**：用户在任何看到数据的地方，都能立即理解它是什么、怎么判断好坏。

**三层说明体系**：

| 层级 | 位置 | 触发方式 | 内容 |
|---|---|---|---|
| 全局字典 | 新页面 `/data-dict` | 导航栏进入 | 完整字段列表，含含义/标准/来源/更新频率 |
| 内联提示 | 表头、卡片 label | hover `?` 图标 | 一句话解释 + 好坏标准 |
| 颜色编码 | 数值旁边 | 自动 | 绿=好/红=差/灰=中性（已有部分） |

**字典内容示例**：

```
ROE（净资产收益率）
  含义：公司每 100 元净资产一年能赚多少钱，衡量资本运用效率
  判断：>15% 优秀 · 5-15% 一般 · <5% 差
  来源：Baostock query_profit_data（季度更新）
  影响：fundamental_score 权重 25%

趋势评分
  含义：基于 MA20/MA60 均线排列的技术面评分（0-100）
  判断：>80 强劲 · 50-80 向好 · <50 弱势
  来源：factors 表，由 calc_factors.py 计算
  影响：总分权重 35%

Sharpe Ratio
  含义：每承担一单位风险获得的超额回报，衡量风险调整后收益
  判断：>1 优秀 · 0.5-1 良好 · <0 较差
  来源：复盘 API 实时计算
```

### 涉及字段清单（共 30+ 个）

**基本面**：PE、PB、ROE、营收、营收增速、净利润、净利润增速、EPS、资产负债率、行业

**技术面**：MA20、MA60、趋势评分、动量评分、流动性评分、风险评分、总评分

**复盘**：Sharpe Ratio、盈亏比、胜率、最大回撤、因子 IC、换手率、行业集中度

**Agent**：AI 评级(1-5)、共识级别、投票数

**行情**：开盘价、最高价、最低价、收盘价、成交量、成交额、换手率

### 实现方案

| 文件 | 说明 |
|---|---|
| `docs/data-dictionary.md` | 完整字典（Markdown，可被 Web 读取） |
| `app/web/app/data-dict/page.tsx` | 字典页面（从 Markdown 渲染） |
| 各页面组件 | 表头/label 旁加 `?` tooltip |

### 验收标准

- 字典页面可访问，搜索过滤
- 关键字段（PE/ROE/Sharpe/因子评分/趋势线）有内联提示
- 颜色编码覆盖所有数值字段

## 14.2 动态日志 & 进度展示

### 背景

用户点击「更新数据」或「运行筛选」后，等待时间长达 60 分钟（更新数据）或 10 分钟（含 Agent 辩论），期间无法知道系统在做什么、进度如何、是否卡住。

### 产品需求

**目标**：点击按钮后，仪表盘实时显示当前执行步骤、进度百分比、已处理/总数。

**展示内容**：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔄 正在采集行情数据
   进度：████████░░░ 67%（2000/2983）
   当前：000001 平安银行 ✓
   耗时：已运行 35 分钟，预计剩余 18 分钟
   失败：2 只（000010、000016）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 技术方案

| 层 | 改动 |
|---|---|
| Worker | 每个脚本在关键步骤写 `system_tasks` 表（`step_detail` JSON 字段） |
| API | `GET /api/tasks/:id` 返回 `step_detail` |
| 前端 | 仪表盘轮询展示进度条 |

**Worker 需要改的脚本**：

| 脚本 | 写入时机 | 内容 |
|---|---|---|
| `update_stocks.py` | 采集完成后 | `{"step": "stocks", "done": true, "count": 5204}` |
| `update_prices.py` | 每 100 只后 | `{"step": "prices", "current": 2000, "total": 2983, "failed": ["000010"]}` |
| `calc_factors.py` | 开始/结束 | `{"step": "factors", "done": true}` |
| `run_screening.py` | 开始/结束 | `{"step": "screening", "done": true}` |
| `update_watchlist.py` | 开始/结束 | `{"step": "watchlist", "done": true}` |
| `generate_report.py` | 每只股票后 | `{"step": "ai", "current": 3, "total": 5, "mode": "committee"}` |

**数据库改动**：`system_tasks` 表已有字段，不需要新增列。`step_detail` 存为 TEXT（JSON 字符串），`error_message` 长度足够存摘要。

### 前端 UI

仪表盘「任务链路」区域改为实时进度：

- 当前步骤高亮 + 脉冲动画
- 已完成步骤绿色 ✓
- 未开始步骤灰色
- 失败步骤红色 ✗ + 错误信息

### 验收标准

- 点击按钮后 2 秒内仪表盘显示进度
- 每一步完成/失败有明确标识
- 进度条实时更新（2 秒轮询）
- 失败时显示原因
