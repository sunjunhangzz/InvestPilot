# 04-观察池

## 任务 04.1：实现观察池进入规则

所属子项目：worker

前置依赖：03.4

开发内容：

- 创建 `worker/scripts/update_watchlist.py`
- 将 A/B 类推荐加入观察池。
- 已存在股票更新最新推荐日期和价格。
- 保留首次推荐日期。

验收标准：

- `watchlist` 有数据。
- 重复推荐不会覆盖首次推荐日期。

风险点：

- 同一股票重复插入。

## 任务 04.2：实现跟踪天数更新

所属子项目：worker

前置依赖：04.1

开发内容：

- 按交易日计算跟踪天数。
- 更新最新价格。
- 计算推荐后收益。

验收标准：

- 已跟踪天数正确。
- 非交易日不错误增加天数。

风险点：

- 交易日和自然日混淆。

## 任务 04.3：实现退出和降级规则

所属子项目：worker

前置依赖：04.2

开发内容：

- 跌破 MA20 或 MA60 时标记。
- 流动性明显恶化时标记。
- 达到最大观察周期后退出。
- 退出必须记录原因。

验收标准：

- 状态流转正确。
- 不会无原因移除股票。

风险点：

- 退出逻辑过于激进。
