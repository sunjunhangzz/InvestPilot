## 任务 01.3：实现任务状态写入

所属子项目：worker

前置依赖：01.2

开发内容：

- 提供 worker 更新 `system_tasks` 的工具函数。
- 支持状态：
  - pending
  - running
  - success
  - failed
  - cancelled
- 明确 API 和 worker 的职责边界：
  - API 创建任务并写入 pending。
  - worker 接手任务后写入 running。
  - worker 完成后写入 success 或 failed。

验收标准：

- 能写入一条任务记录。
- 能更新任务状态。
- 失败时能写入错误信息。
- 不同脚本使用同一套任务状态工具。

风险点：

- 后续每个脚本如果不统一使用，会导致任务状态不一致。

## 任务 01.3a：实现任务日志摘要规范

所属子项目：worker / web

前置依赖：01.3

开发内容：

- 规定 `system_tasks.error_message` 只保存错误摘要。
- 详细错误写入 `data/logs/`。
- 任务成功时记录 started_at 和 finished_at。

验收标准：

- 前端能看到失败摘要。
- 本地日志能看到详细堆栈或失败上下文。

风险点：

- 把大量错误堆栈塞进数据库字段，影响可读性。
