# 08-本地联调

## 任务 08.1：完整手动流程联调

所属子项目：全局

前置依赖：01-07 全部完成

联调流程：

```text
init_db
update_stocks
update_prices
calc_factors
run_screening
update_watchlist
generate_report
网页查看
```

API 组合任务联调流程：

```text
POST /api/tasks/update-data
轮询 task 状态
POST /api/tasks/run-screening
轮询 task 状态
GET /api/runs/latest
GET /api/recommendations/today
GET /api/watchlist
网页查看
```

验收标准：

- 完整流程能跑通。
- 数据库有推荐结果。
- 网页能显示推荐和观察池。

## 任务 08.2：异常场景联调

所属子项目：全局

前置依赖：08.1

测试场景：

- 无 API Key
- AI 调用失败
- 无网络
- 非交易日
- 重复点击任务按钮
- 数据源返回空
- SQLite 写入冲突

验收标准：

- 系统有明确错误提示。
- 基础推荐不因 AI 失败中断。
- 旧数据不被空数据覆盖。

## 任务 08.4：健康检查脚本

所属子项目：worker

前置依赖：01.5

开发内容：

- 创建 `scripts/health_check.py`。
- 检查 Python 版本。
- 检查依赖是否可导入。
- 检查数据库是否存在。
- 检查核心表和关键字段。

验收标准：

- 一条命令能输出环境健康状态。
- 缺依赖、缺表、缺字段时能明确提示。

- 没有健康检查时，联调问题难定位。

## 任务 08.4a：日志检查

所属子项目：全局

前置依赖：00.7

开发内容：

- 检查 `data/logs/` 是否存在。
- 检查 worker 是否能写日志。
- 检查 API 是否能写日志。
- 检查任务失败时是否同时写入 `system_tasks` 和本地日志。

验收标准：

- 任务失败能在前端看到摘要。
- 本地日志能看到详细错误。
- 日志文件不会进入 Git。

风险点：

- 只有数据库摘要，没有详细日志，问题无法定位。

## 任务 08.5：一键流程脚本

所属子项目：worker

前置依赖：08.1

开发内容：

- 创建 `scripts/run_pipeline.py`。
- 串行执行：
  - update_stocks
  - update_prices
  - calc_factors
  - run_screening
  - update_watchlist
  - generate_report

验收标准：

- 一条命令能跑完整主链路。
- 某一步失败时停止并输出原因。
- 失败不会破坏已有成功数据。

风险点：

- 一键脚本掩盖各步骤错误，需要保留详细日志。
