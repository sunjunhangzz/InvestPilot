# 02-数据采集

## 任务 02.1：实现股票列表更新

所属子项目：worker

前置依赖：01.2

开发内容：

- 创建 `worker/scripts/update_stocks.py`
- 使用 AkShare 获取 A股股票基础列表。
- 写入 `stocks` 表。
- 标记主板、ST、是否有效。

验收标准：

- `stocks` 表有数据。
- 能筛出主板股票。
- 重复运行不重复插入。

风险点：

- AkShare 字段变化。
- 主板判断规则需要根据实际数据修正。

## 任务 02.1a：股票列表原始数据落盘

所属子项目：worker

前置依赖：01.2

开发内容：

- 先把 AkShare 返回的股票列表原始数据保存到 `data/raw/`。
- 保留采集日期和来源信息。

验收标准：

- 能看到原始数据文件。
- AkShare 字段变化时可以回看原始结构。

风险点：

- 直接入库会掩盖字段变化问题。

## 任务 02.1b：主板股票池识别

所属子项目：worker

前置依赖：02.1a

开发内容：

- 根据股票代码、市场、名称等字段识别主板。
- 输出主板股票数量。
- 输出被排除股票数量和原因。

验收标准：

- 主板识别规则可解释。
- 主板数量大致合理。

风险点：

- 主板、创业板、科创板、北交所判断错误。

## 任务 02.2a：单只股票行情拉取测试

所属子项目：worker

前置依赖：02.1b

开发内容：

- 先选择 1-3 只主板股票测试行情拉取。
- 检查字段、日期、复权、成交额。

验收标准：

- 单只股票能成功写入 `daily_prices`。
- 数据字段符合数据库设计。

风险点：

- 直接批量拉取时错误难定位。

## 任务 02.2：实现行情更新

所属子项目：worker

前置依赖：02.1

开发内容：

- 创建 `worker/scripts/update_prices.py`
- 获取主板股票最近约 120 个交易日日线。
- 优先使用前复权数据。
- 写入 `daily_prices` 表。

验收标准：

- `daily_prices` 有数据。
- 每只股票至少有足够计算 MA60 的数据。
- 空数据不覆盖旧数据。
- 失败股票记录日志。

风险点：

- 全量获取耗时长。
- AkShare 限流或返回空数据。

## 任务 02.2b：批量行情失败重试

所属子项目：worker

前置依赖：02.2

开发内容：

- 对失败股票记录失败原因。
- 支持跳过失败股票继续处理。
- 支持后续重试失败股票。

验收标准：

- 单只股票失败不影响整个批次。
- 日志能看到失败股票列表。

风险点：

- 批量任务中断导致已写入和未写入状态不清。

## 任务 02.3：处理非交易日和最近交易日

所属子项目：worker

前置依赖：02.2

开发内容：

- 根据 `daily_prices` 推断最近可用交易日。
- 周末或节假日运行时使用最近交易日。

验收标准：

- 非交易日运行不报错。
- 推荐基于最近可用交易日。

风险点：

- 交易日历不准确。
