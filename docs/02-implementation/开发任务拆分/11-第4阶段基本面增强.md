# 11-第4阶段基本面增强

> 2026-07 经 5 轮多角度方案评估 + 4 个数据源实测后确定。

## 方案概要

原阶段四计划 8 项数据增强。经 AkShare / Baostock / Efinance / mootdx 四个数据源实测后：

| 原计划 | 可行 | 数据源 | 原因 |
|---|---|---|---|
| 基本面（ROE/EPS/PE/PB/市值） | ✅ | Efinance + Baostock | 批量获取 5500+ 只 |
| 财报（资产/负债/现金流） | ✅ | Baostock balance+cashflow | 按报告期查询 |
| 行业分类 | ✅ | Efinance base_info + Baostock industry | 申万行业 |
| 成长性（营收/利润增速） | ✅ | Efinance performance batch | 同比增长率 |
| 公告数据 | ❌ | — | 被代理阻断 |
| 新闻/情绪 | ❌ | — | 被代理阻断 |
| 重大合同/合作 | ❌ | — | 需 NLP 解析 |
| 政策事件 | ❌ | — | 无现成 API |

**数据源**：

| 数据源 | 用途 | 特点 |
|---|---|---|
| Efinance | PE/PB/ROE/利润/营收/市值/行业 | 批量 5500+ 只，一次请求 |
| Baostock | 资产负债/现金流/杜邦分析 | 按报告期，单只查询 |
| mootdx | 深度财务数据 | 通达信数据，补充字段 |

## 增强层原则

- 有数据就加分或扣分
- 没数据不影响核心推荐
- 数据异常不阻塞系统
- 基本面不参与硬过滤，只做加分/扣分

## 评分策略

| 指标 | 好 | 差 | 加分/扣分 | 权重 |
|---|---|---|---|---|
| ROE | > 15% | < 5% | +10 / -5 | 25% |
| 营收增速 YoY | > 20% | < 0% | +10 / -5 | 25% |
| 净利润增速 YoY | > 20% | < -30%（暴跌） | +10 / -10 | 25% |
| PE/PB 分位 | 行业中位以下 | 行业中位 2 倍以上 | +5 / -5 | 15% |
| 资产负债率 | < 60% | > 80% | +5 / -5 | 10% |

基本面总分 = 各指标加权和，加到现有总评分上（增幅 0-10 分）。

## 任务拆分

### 任务 11.1：基本面数据采集

所属子项目：worker

前置依赖：02（数据采集已完成）

**开发内容**：

- 安装 `efinance` + `baostock` 依赖
- 创建 `app/worker/src/data_sources/fundamental.py`
  - `fetch_financials()` — 从 Efinance 批量获取全市场基本面
  - `fetch_balance()` — Baostock 资产负债/现金流
  - `fetch_industry()` — Efinance 行业分类
- 新增表 `fundamentals`：

```sql
CREATE TABLE fundamentals (
  code TEXT PRIMARY KEY,
  pe REAL, pb REAL, roe REAL, market_cap REAL,
  revenue REAL, revenue_yoy REAL,
  net_profit REAL, net_profit_yoy REAL,
  eps REAL, debt_ratio REAL,
  industry TEXT,
  updated_at TEXT
);
```

- 创建 `app/worker/scripts/update_fundamentals.py`
- 数据写入 UPSERT（不支持覆盖旧有效数据）

**验收标准**：

- `fundamentals` 表有 5000+ 行数据
- PE/PB/ROE/营收/利润/行业字段不为空

### 任务 11.2：基本面评分

所属子项目：worker

前置依赖：11.1

**开发内容**：

- 在 `app/worker/src/factors/scoring.py` 新增 `fundamental_score(f) → float`
- 在 `calc_factors.py` 中 JOIN fundamentals 表，计算基本面得分
- 总分 = 原总分 + fundamental_score × 0.1（增幅系数）

**验收标准**：

- 有基本面数据的股票总分有调整
- 无基本面数据的股票不受影响（原始分数不变）

---

### 任务 11.3：行业分类标注

所属子项目：worker

前置依赖：11.1（已在 11.1 中一并实现）

**开发内容**：

- 将 Efinance 行业写入 `stocks.industry` 字段（此前为 NULL）
- 推荐列表中显示行业名称

**验收标准**：

- `stocks.industry` 非空比例 > 90%
- 今日推荐列表显示行业

---

### 任务 11.4：基本面 Web 展示

所属子项目：web

前置依赖：11.2

**开发内容**：

**验收标准**：
