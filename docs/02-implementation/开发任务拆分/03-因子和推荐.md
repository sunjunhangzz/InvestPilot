# 03-因子和推荐

## 任务 03.1：计算基础技术指标

所属子项目：worker

前置依赖：02.2

开发内容：

- 计算 MA20。
- 计算 MA60。
- 计算 20 日收益率。
- 计算 60 日收益率。
- 计算近 20 日日均成交额。
- 计算近 20 日波动率。
- 计算近 20 日最大回撤。

验收标准：

- 指标计算结果可写入中间结构。
- 缺少历史数据的股票跳过。

风险点：

- 未来函数。
- 复权数据不一致。

## 任务 03.2：实现因子评分

所属子项目：worker

前置依赖：03.1

开发内容：

- 趋势评分
- 动量评分
- 流动性评分
- 风险评分
- 总评分
- 写入 `factors`

验收标准：

- `factors` 有当前 run_id 的评分。
- 分数范围稳定，例如 0-100。
- 总分排序合理。

风险点：

- 极端值影响评分。
- 不同行业差异未处理，第一版可接受。

## 任务 03.3：实现筛选规则

所属子项目：worker

前置依赖：03.2

开发内容：

- 排除非主板。
- 排除 ST。
- 排除上市不足 180 天。
- 排除低流动性。
- 排除价格过低或过高。
- 应用趋势条件。

验收标准：

- 候选股票数量合理。
- 明显不符合条件的股票不会进入推荐。

风险点：

- 过滤过严导致推荐不足 10 只。

## 任务 03.4：生成推荐结果

所属子项目：worker

前置依赖：03.3

开发内容：

- 创建 `worker/scripts/run_screening.py`
- 生成 A/B/C 分层。
- 写入 `recommendations`。

验收标准：

- 推荐数量在 10-50。
- 每条推荐有 `run_id`。
- 每条推荐有 reason 和 risk_tags。

风险点：

- reason 第一版可能较模板化，可接受。

## 任务 03.5：生成推荐理由和风险标签

所属子项目：worker

前置依赖：03.4

开发内容：

- 根据规则生成 `reason`。
- 根据风险条件生成 `risk_tags`。
- 不依赖 AI。

示例：

```text
入选理由：站上 MA20 和 MA60，20日收益为正，成交额满足要求。
风险标签：短期涨幅偏高 / 波动率偏高 / 流动性一般。
```

验收标准：

- 每条推荐都有 reason。
- 高风险股票有 risk_tags。
- 没有 AI 时仍能解释推荐原因。

风险点：

- 推荐理由过于模板化，但第一版可接受。
