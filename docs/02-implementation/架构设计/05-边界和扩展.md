# 05-边界和扩展

## 第一版边界

第一版只做核心层：

- 主板股票池
- 日线行情
- 基础趋势筛选
- 因子评分
- 推荐列表
- 观察池
- 基础 AI 解释
- 本地网页展示

## 增强层扩展

后续可以加入：

- 财报数据
- 公告数据
- 重大合同识别
- 新闻数据
- 情绪数据
- 政策事件
- 微信推送
- 早盘和尾盘定时报告

扩展原则：

- 增强层失败不能影响核心推荐。
- 没有增强数据时，核心推荐照常输出。
- 增强数据只加分、扣分或提示风险，不直接替代核心策略。

## 实验层扩展

后续可以加入：

- 多 Agent 专家委员会
- Agent 自我复盘
- 策略优化算法
- 贝叶斯优化
- 遗传算法
- 多目标优化

扩展原则：

- 先影子运行。
- 不直接覆盖正式策略。
- 需要人工确认后才生效。
- 所有配置需要版本化。

## 什么时候拆独立后端

第一版不拆独立后端。

当出现以下情况时再拆：

- 任务队列复杂
- 多用户权限需求
- 需要部署服务器
- SQLite 不够用
- 需要 PostgreSQL / Redis
- 需要长期后台任务

拆分后可能结构：

```text
web: Next.js
api: Fastify / NestJS
worker: Python
queue: Redis / BullMQ
db: PostgreSQL
```

## 什么时候换数据库

SQLite 适合第一版。

需要换数据库的信号：

- 多进程频繁写入
- 数据量增长明显
- 多用户访问
- 需要远程部署
- 查询性能明显下降

候选：

- DuckDB：适合分析型本地数据
- PostgreSQL：适合长期服务化
