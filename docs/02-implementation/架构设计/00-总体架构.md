# 00-总体架构

## 架构目标

第一版架构目标：

- 本地单机可运行
- 低运维成本
- 数据链路稳定
- 任务可追踪
- 结果可解释
- 后续可扩展到多维度数据和多 Agent

## 总体分层

```text
用户浏览器
  ↓
app/web
  - 页面
  - API Routes
  - 任务创建和状态查询
  ↓
app/worker
  - 数据采集
  - 因子计算
  - 推荐生成
  - 观察池维护
  - AI报告生成
  ↓
data/a_stock_research.sqlite
  - 股票数据
  - 行情数据
  - 因子
  - 推荐
  - 观察池
  - 任务状态
  ↓
data/logs/
  - worker日志
  - 数据源日志
  - AI日志
  - API日志
  ↓
reports/
  - 报告文件
```

## 子项目职责

```text
app/web
  用户界面、API Routes、任务触发、数据读取

app/worker
  Python 数据和计算任务

app/shared
  公共配置、字段约定、运行参数

app/api
  后续独立后端预留，第一版不开发
```

## 第一版不做

第一版不引入：

- 独立后端服务
- PostgreSQL
- Redis
- 消息队列
- Docker
- 多用户权限
- 自动交易
- 真正多 Agent 编排

## 关键架构原则

```text
前端不直接调用 Python
web API Routes 负责创建任务
worker 负责写数据库
web 负责读数据库
shared 负责统一配置和字段约定
SQLite 写入任务必须串行
AI 失败不能影响基础推荐
system_tasks 保存任务摘要
data/logs 保存详细日志
```

## 运行方式

第一版本地运行：

```text
启动 Next.js web
  ↓
用户在网页点击任务按钮
  ↓
API Routes 启动 worker 脚本
  ↓
worker 写 SQLite
  ↓
网页读取 SQLite 结果
```
