# 分阶段路线图

## 第1阶段：核心MVP

目标：跑通每天推荐主链路。

交付内容：

- 本地 Next.js 网站
- SQLite 数据库
- A股主板股票池
- 日线行情更新
- 趋势、动量、流动性、风险评分
- 10-50 只推荐股票
- 观察池
- 基础 AI 解释报告

## 第2阶段：报告和使用体验

目标：让系统每天可用、可读、可复查。

交付内容：

- 首页仪表盘
- 今日推荐页
- 股票详情页
- 观察池页
- 报告中心
- 系统设置
- 任务日志
- 错误提示

## 第3阶段：自动化和推送

目标：减少手动操作。

交付内容：

- 08:30 早盘前报告
- 14:00 尾盘前报告
- 定时任务
- 微信推送配置
- 推送成功和失败记录
- 报告重发功能

## 第4阶段：增强数据

目标：从单纯趋势扩展到多维度投研。

交付内容：

- 基本面数据
- 财报数据
- 公告数据
- 重大合同和合约识别
- 合作意向和战略合作识别
- 政策事件
- 新闻数据
- 情绪数据

## 第5阶段：复盘闭环

目标：验证推荐是否真的有效。

交付内容：

- 昨日推荐复盘
- 前一周推荐复盘
- 前一个月推荐复盘
- 推荐后 1日、1周、2周、4周收益
- 最大回撤
- 趋势破位统计
- 行业维度表现
- 因子维度表现

## 第6阶段：多Agent专家委员会

目标：让 AI 做最后一轮多视角评审和排雷。

Agent 角色：

- 基本面分析师
- 财报分析师
- 技术面分析师
- 消息面分析师
- 情绪面分析师
- 政策面分析师
- 事件催化分析师
- 行业研究员
- 风险官
- 投资委员会主席

## 第7阶段：策略优化

目标：基于历史表现改进策略。

交付内容：

- 策略版本管理
- 因子权重优化
- 阈值优化
- 网格搜索
- 随机搜索
- 滚动窗口验证
- 优化建议人工确认

## 第8阶段：高级优化和Agent自我复盘

目标：验证高级实验能力。

交付内容：

- 贝叶斯优化
- 遗传算法
- 多目标优化
- Agent 表现复盘
- Agent 提示词优化
- Agent 权重优化
- 风险官否决规则优化

## 阶段推进原则

- 每个阶段都必须可独立交付。
- 后一阶段不能破坏前一阶段稳定性。
- 增强功能失败不能影响核心推荐。
- 实验功能必须先影子运行。
- 所有策略变化必须版本化。
