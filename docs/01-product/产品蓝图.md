# 产品蓝图

## 长期定位

长期系统定位：

```text
A股主板稳健趋势股多维度 AI 投研助手
```

系统长期目标是每天在固定时间从 A股主板中筛选 10-50 只稳健趋势候选股，并结合行情、基本面、财报、公告、重大合同、消息面、情绪面、政策面、资金流和风险事件生成投资参考报告。

系统不做自动交易，不直接下单。

## 三层架构

系统能力分为三层：

```text
核心层：必须稳定，决定每天能不能正常推荐
增强层：提高质量，失败不能影响核心推荐
实验层：探索优化，默认不影响正式结果
```

## 核心层

核心层是正式推荐的基础。

包含：

- A股主板股票池
- 行情数据
- 趋势因子
- 动量因子
- 流动性因子
- 风险扣分
- 观察池
- 基础 AI 解释报告

核心层必须简单、稳定、可解释、可复盘。

## 增强层

增强层用于提高推荐质量，但失败时不能阻塞核心推荐。

包含：

- 财报质量分析
- 基本面分析
- 公告分析
- 重大合同分析
- 业务合约分析
- 合作意向分析
- 战略合作分析
- 政策面分析
- 消息面分析
- 情绪面分析
- 资金流分析
- 微信推送
- 08:30 早盘前报告
- 14:00 尾盘前报告

增强层原则：

- 有数据就加分或扣分。
- 没数据不影响核心推荐。
- 数据异常不阻塞系统。
- AI 分析失败不影响基础列表。

## 实验层

实验层用于验证高级能力，默认不影响正式推荐。

包含：

- 真正多 Agent 专家委员会
- Agent 自我复盘
- Agent 提示词优化
- Agent 权重优化
- 风险官否决规则优化
- 策略优化算法
- 贝叶斯优化
- 遗传算法
- 多目标优化
- 情绪模型
- 自动调参

实验层原则：

- 只生成建议，不直接改正式策略。
- 默认影子运行。
- 需要人工确认后才能应用。
- 保留版本记录。
- 不允许使用未来数据。

## 多维度评估

长期评估维度：

- 技术趋势
- 动量
- 流动性
- 基本面
- 财报质量
- 消息面
- 情绪面
- 政策面
- 公告事件
- 重大合同
- 业务合约
- 合作意向
- 战略合作
- 资金流
- 风险事件

重大合同、业务合约、战略合作、合作意向可以作为加分项，但需要判断：

- 真实性
- 金额大小
- 占营收比例
- 履约周期
- 持续性
- 是否会实质影响业绩

如果基本面、财报或风险公告存在明显硬伤，即使技术走势较好，也应降级或剔除。

## 报告体系

长期报告类型：

- 早盘前报告：08:30 左右
- 尾盘前报告：14:00 左右
- 个股报告
- 每日汇总报告
- 周度复盘报告
- 月度复盘报告
- 策略优化报告
- Agent 表现复盘报告

## 微信推送

长期支持报告推送到微信。

优先方案：

- 企业微信群机器人（无限制、Markdown 排版、腾讯官方，当前默认）
- PushPlus（200 条/天，备选）
- Server酱（5 条/天，备选）

不优先使用个人微信自动化登录方案，因为稳定性、封号风险和维护成本更高。
