# 项目规划

## 第一阶段：本地 MVP

目标：跑通从 A股主板数据获取到稳健趋势股推荐报告的完整链路。

功能范围：

- 创建本地网站。
- 创建本地数据库。
- 用 Python 获取 A股行情数据。
- 建立 A股主板股票池。
- 剔除 ST、退市风险、停牌、流动性过低和明显财务异常股票。
- 用趋势规则、基本面、财报、消息面、情绪面、政策面和事件催化因子筛选候选股票。
- 调用 AI 生成简版个股分析。
- 输出每日推荐列表和报告。
- 支持 08:30 左右生成早盘前报告。
- 支持 14:00 左右生成尾盘前报告。
- 支持报告生成后推送到微信。

## 第二阶段：策略和因子增强

目标：提升稳健趋势股筛选质量。

计划加入：

- 趋势因子
- 动量因子
- 量价因子
- 波动率因子
- 基本面质量因子
- 财报质量因子
- 消息面因子
- 情绪面因子
- 政策面因子
- 事件催化因子
- 流动性因子
- 主板风险过滤因子
- 风险扣分因子

优先级：

1. 趋势确认
2. 流动性充足
3. 基本面和财报没有明显硬伤
4. 消息面、情绪面、政策面存在合理支撑
5. 重大合同、合约、合作意向等事件催化真实可验证
6. 短期不过热
7. 风险事件可控

## 第三阶段：AI 多 Agent 专家委员会

目标：让 AI 不直接荐股，而是做最后一轮评审和排雷，重点检查趋势是否健康、基本面和财报是否存在硬伤、消息和情绪是否过热、政策和事件催化是否真实可验证。

Agent 角色：

- 基本面分析师
- 财报分析师
- 消息面分析师
- 技术面分析师
- 情绪面分析师
- 政策面分析师
- 事件催化分析师
- 行业研究员
- 风险官
- 投资委员会主席

## 第四阶段：回测和复盘

目标：验证系统是否真的有参考价值。

重点指标：

- 1周收益
- 2周收益
- 4周收益
- 胜率
- 最大回撤
- 平均回撤
- 推荐后是否跌破趋势线
- 行业分布
- 推荐后表现复盘

复盘周期：

- 昨日推荐
- 前一周推荐
- 前一个月推荐

复盘目标：

- 判断推荐后是否上涨
- 判断推荐后最大回撤是否可控
- 判断推荐后是否跌破趋势线
- 判断推荐股票在一周观察期内是否仍保持趋势
- 判断哪些因子有效
- 判断哪些行业和市场环境下推荐质量更高
- 为下一轮策略参数优化提供依据

## 第五阶段：自动化和推送

目标：减少手动操作，让系统按固定时间生成报告并发送到微信。

计划加入：

- 08:30 早盘前报告任务
- 14:00 尾盘前报告任务
- 微信推送配置
- 推送成功和失败记录
- 报告重发功能

## 第六阶段：AI复盘和策略优化

目标：让系统每天检查自己的推荐表现，并结合优化算法生成策略优化建议。

计划加入：

- 每日推荐表现复盘
- 前一周推荐表现复盘
- 前一个月推荐表现复盘
- 推荐观察池维护
- 推荐股票最少跟踪 5 个交易日
- AI 归因分析
- 因子权重优化建议
- 风险过滤规则优化建议
- 策略参数调整建议
- 网格搜索 / 随机搜索
- 滚动窗口验证
- 贝叶斯优化
- 多目标优化
- AI Agent 专家委员会复盘
- Agent 提示词优化
- Agent 权重优化
- 风险官否决规则优化
- 人工确认后再应用策略调整

优化算法原则：

- AI 负责解释问题和提出方向
- 算法负责参数搜索和效果验证
- 人负责确认是否应用
- 不允许使用未来数据
- 不允许只按收益单一指标优化

AI Agent 优化原则：

- 每日检查明显误判案例
- 每周评估各 Agent 观点质量
- 每月调整 Agent 配置
- 所有 Agent 配置需要版本化
- 新配置需要人工确认后生效
