# 使用原则

## 不自动交易

系统只输出股票池和投研报告，不直接连接券商账户，不自动下单。

系统只做投资参考，不替代人工判断。

## AI 只做辅助判断

AI 的作用是解释、总结、排雷和交叉验证，不直接决定买卖。

## 数据优先于观点

系统优先使用结构化行情、财务、资金和公告数据，再让 AI 做解释。

AI 输出必须能被数据验证，不能只依赖模型观点。

## 多维度评估

系统推荐股票时必须综合多个维度，不只看技术走势。

核心维度包括：

- 技术趋势
- 基本面
- 财报质量
- 消息面
- 情绪面
- 政策面
- 公告事件
- 重大合同、业务合约和合作意向
- 资金流和流动性
- 风险事件

重大合同、业务合约、战略合作、合作意向可以作为加分项，但需要判断真实性、金额大小、履约周期、持续性，以及是否会实质影响业绩。

如果基本面、财报或风险公告存在明显硬伤，即使技术走势较好，也应降级或剔除。

## 每日固定时间运行

系统支持每天两个关键时间点生成报告：

- 08:30 左右：早盘前报告
- 14:00 左右：尾盘前报告

早盘前报告主要用于开盘前查看，尾盘前报告主要用于尾盘决策参考。

系统关注 1-4 周趋势机会，不做盘中追涨杀跌。

## 微信推送

系统报告生成后，可以推送到微信，方便在手机上查看。

第一版优先考虑稳定、低维护的推送方式，例如企业微信机器人、PushPlus、Server酱 或 WxPusher。

不建议第一版使用个人微信自动化登录方案，因为稳定性、封号风险和维护成本都更高。

电脑登录微信不等于系统可以稳定自动发送微信消息。

个人微信自动化通常依赖桌面控制、插件或非官方协议，维护成本高，也存在稳定性和账号风险。

推荐优先使用扫码绑定或 Token 推送方式：

- 企业微信群机器人（免费无限制、Markdown 排版，当前默认）
- PushPlus（200 条/天）
- Server酱（5 条/天）
- WxPusher

## AI调用方式

系统优先使用 API 调用大模型，因为 API 适合定时任务、自动报告、错误重试和日志记录。

ChatGPT Plus 订阅不等于 OpenAI API 额度。OpenAI API 需要单独开通计费。

系统应支持多种 AI Provider：

- DeepSeek API
- 通义千问 API
- 智谱 GLM API
- Kimi API
- OpenAI API
- Ollama 本地模型
- CLI / 手动辅助模式

CLI / 手动辅助模式只作为备用，不作为第一版定时报告的首选方案。

## 只做A股主板

第一版系统只覆盖 A股主板股票。

默认排除：

- ST 和退市风险股票
- 停牌股票
- 流动性过低股票
- 明显财务异常股票
- 短期涨幅过高且情绪过热股票
- 重大负面公告股票
- 监管问询和重大诉讼风险股票

## 稳健趋势优先

系统优先寻找趋势已经确认、成交活跃、基本面没有明显硬伤的股票。

系统不追求连板股、妖股、极端题材炒作和高波动博弈。

## 推荐连续性

每日推荐股票需要进入观察池，最少跟踪 5 个交易日。

系统不应该今天推荐、明天无理由移除。除非触发明确风险退出条件，否则推荐股票应持续跟踪并更新状态。

观察状态包括：

- 正在观察
- 持续观察
- 降级观察
- 退出观察
- 风险阻断

退出观察必须记录原因。

## 控制仓位

即使系统筛出候选股票，也不建议单只股票仓位过高。

个人小资金账户更应该控制风险，避免满仓、追高和频繁交易。

即使系统给出重点观察股票，也应结合大盘环境、板块状态和个人仓位再决策。
