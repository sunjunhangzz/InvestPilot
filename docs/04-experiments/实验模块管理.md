# 实验模块管理

## 目标

实验模块用于验证高级能力，但默认不影响正式推荐。

适用模块：

- 多 Agent 专家委员会
- Agent 自我复盘
- Agent 提示词优化
- 策略优化算法
- 贝叶斯优化
- 遗传算法
- 多目标优化
- 复杂情绪分析

## 影子运行

实验模块必须先影子运行。

影子运行含义：

```text
正式推荐仍由核心策略产生
实验模块同时生成自己的候选结果
系统记录实验结果
实验结果不直接展示为正式推荐
```

示例：

```text
正式推荐：核心策略 v1
影子推荐：多Agent策略 v1
影子推荐：优化算法策略 v1
```

运行一段时间后比较：

- 1周收益
- 2周收益
- 4周收益
- 胜率
- 最大回撤
- 趋势破位率
- 推荐稳定性
- 行业集中度

## 人工确认

实验模块不能自动覆盖正式策略。

以下变更必须人工确认：

- 因子权重变化
- 策略阈值变化
- A类/B类评级分界线变化
- Agent 提示词变化
- Agent 权重变化
- 风险官否决规则变化
- 推荐数量变化

## 版本管理

所有实验配置必须版本化。

需要记录：

- 策略版本
- Prompt 版本
- Agent 配置版本
- 因子权重版本
- 优化算法版本
- 生效时间
- 变更原因
- 复盘证据

## 禁止事项

实验模块禁止：

- 使用未来数据。
- 只按收益单一指标优化。
- 自动覆盖线上策略。
- 根据单日结果调整配置。
- 在数据不完整时强行给出优化结论。
- 让 AI 直接决定买卖。

## 升级为正式模块的条件

实验模块满足以下条件后，才可以考虑进入正式链路：

- 至少运行一个完整观察周期。
- 表现优于核心策略或明显降低风险。
- 最大回撤不高于核心策略。
- 推荐稳定性可接受。
- 结果可解释。
- 用户人工确认。
