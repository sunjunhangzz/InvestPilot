# 04-日志规范

## 日志目标

所有失败必须能定位。

第一版日志分两层：

```text
system_tasks    任务摘要，给前端看
data/logs/      详细日志，给开发排查
```

## 日志目录

```text
data/logs/
```

建议文件：

```text
worker.log
data_source.log
screening.log
ai.log
api.log
```

## 日志格式

使用 JSON Lines。

每行一条日志。

必须字段：

```text
time
level
module
message
```

尽量带上：

```text
task_id
run_id
trade_date
context
```

示例：

```json
{
  "time": "2026-07-04T09:00:00+08:00",
  "level": "INFO",
  "module": "update_prices",
  "task_id": "task_20260704_001",
  "run_id": "20260704-manual-001",
  "message": "start update prices",
  "context": {
    "stock_count": 1200
  }
}
```

## 日志等级

```text
DEBUG
INFO
WARN
ERROR
```

## 记录要求

必须记录：

- 任务开始
- 任务结束
- 任务失败
- 数据源返回空
- 失败股票代码
- 推荐数量
- 观察池状态变化
- AI 调用失败
- API 子进程退出码

## 禁止事项

日志中禁止写入：

- API Key
- 账户密码
- 任何敏感密钥

## Git规范

必须在 `.gitignore` 排除：

```text
data/logs/
```
