# 06-文档维护规范

## 文档优先级

项目文档按层级阅读：

```text
README.md
docs/README.md
docs/00-overview/
docs/01-product/
docs/02-implementation/
docs/03-operations/
docs/04-experiments/
docs/05-standards/
app/*/docs/
```

## 修改代码前

如果改动涉及：

- 表结构
- API 接口
- 任务流程
- 因子逻辑
- 推荐规则
- 日志格式
- 配置字段

必须先更新对应文档。

## 文档和代码冲突

如果文档和代码冲突：

1. 先确认真实需求。
2. 更新文档。
3. 再改代码。

## 文档分类

产品需求放：

```text
docs/01-product/
```

实现设计放：

```text
docs/02-implementation/
```

运行机制放：

```text
docs/03-operations/
```

实验能力放：

```text
docs/04-experiments/
```

开发规范放：

```text
docs/05-standards/
```

子项目局部说明放：

```text
app/*/docs/
```

## 命名规范

顶层文档可以使用中文文件名。

代码相关文件优先使用英文文件名。

阶段性文档建议带数字前缀，保证阅读顺序稳定。
