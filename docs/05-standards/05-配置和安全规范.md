# 05-配置和安全规范

## 配置来源

公共配置放在：

```text
app/shared/config.json
```

敏感配置放在：

```text
.env
```

示例配置放在：

```text
.env.example
```

## 禁止硬编码

禁止在代码中硬编码：

- API Key
- 数据库路径
- 报告路径
- 日志路径
- 模型密钥

## 路径解析

`app/shared/config.json` 中的相对路径统一相对项目根目录解析。

项目根目录是：

```text
A股AI投研系统/
```

示例：

```json
{
  "databasePath": "data/a_stock_research.sqlite",
  "reportsPath": "reports",
  "logsPath": "data/logs"
}
```

禁止直接用当前工作目录解析这些路径。

web 和 worker 都必须通过统一工具函数解析项目根路径。

## API Key

API Key 只能放在 `.env`。

`.env` 不允许提交到 Git。

`.env.example` 只写变量名，不写真实值。

## AI安全

AI 只做解释和排雷，不直接决定买卖。

AI 调用失败不能影响基础推荐。

AI 输入必须是结构化摘要，不能直接传大量原始行情。

## 数据安全

第一版所有数据本地保存。

不要把数据库、日志、缓存、报告误提交到远程仓库。

`.gitignore` 必须排除：

```text
data/*.sqlite
data/cache/
data/raw/
data/logs/
reports/
.env
.venv/
node_modules/
```
