# 03-数据库规范

## 适用范围

适用于 SQLite 数据库：

```text
data/a_stock_research.sqlite
```

## 字段命名

数据库字段统一使用：

```text
snake_case
```

示例：

```text
trade_date
run_id
task_id
created_at
updated_at
```

## 必备标识

任务必须有：

```text
task_id
```

推荐运行必须有：

```text
run_id
```

交易数据必须有：

```text
trade_date
```

## 写入规范

- 写入必须使用事务。
- 同一时间只允许一个 worker 写数据库。
- 重复运行不能直接覆盖历史结果。
- 数据源返回空数据不能删除旧数据。
- 每次推荐必须新建 `run_id`。

## 查询规范

- 推荐页默认读取最近一次成功 `run_id`。
- 股票详情按需读取历史行情。
- 不一次性返回全市场大数据给前端。

## 表结构变更

表结构变更必须：

- 先更新 `数据库设计.md`
- 再更新 `init_db.py`
- 再更新 schema 校验
- 最后更新 API 和前端

## SQLite注意事项

- SQLite 适合第一版本地单机。
- 不适合多进程高并发写入。
- 写任务必须串行。
- 后续如需要服务化，再考虑 PostgreSQL。
